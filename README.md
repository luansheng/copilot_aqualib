# copilot_aqualib

AquaLib is a multi-agent Python framework for scientific research tasks, featuring a three-agent pipeline (Executor, Reviewer, Searcher), RAG-powered information retrieval, and first-class integration with the [ClawBio](https://github.com/kzy599/ClawBio) bioinformatics skill library.

## Features

- **Three-agent pipeline**: Searcher (RAG-powered context retrieval), Executor (skill dispatch), Reviewer (quality assurance)
- **ClawBio skill priority**: ClawBio bioinformatics skills are automatically preferred when relevant
- **RAG retrieval**: LlamaIndex-based document and skill description indexing
- **Extensible skill registry**: Mount external skill libraries at runtime

## Installation

### Quick start (with ClawBio vendor skills included)

```bash
git clone --recursive https://github.com/kzy599/copilot_aqualib.git
cd copilot_aqualib
pip install -e ".[dev]"
```

The `--recursive` flag ensures the `vendor/ClawBio` submodule is cloned along with the main repository, providing real ClawBio skills out of the box.

### Standard install (without vendor submodule)

```bash
git clone https://github.com/kzy599/copilot_aqualib.git
cd copilot_aqualib
pip install -e ".[dev]"
```

## Setup

### 1. Set your OpenAI API key

```bash
export OPENAI_API_KEY="sk-your-openai-key-here"
```

A single OpenAI API key drives both the LLM agents and the RAG embedding model.

### 2. Initialise the workspace

```bash
aqualib init
```

This creates `aqualib_workspace/` with the following structure:

```
aqualib_workspace/
├── work/                      # Intermediate task files
├── results/                   # Final outputs & audit reports
│   └── clawbio_traces/        # ClawBio skill call logs
├── data/                      # Input data & RAG corpus
└── skills/
    └── clawbio/               # Runtime ClawBio mount point
```

## ClawBio Integration

AquaLib uses a **three-tier priority** system for ClawBio skills:

| Priority | Source | Description |
|----------|--------|-------------|
| **1 (highest)** | Runtime mount point | Your custom skills in `aqualib_workspace/skills/clawbio/` |
| **2** | Vendor directory | `vendor/ClawBio` (included via `git clone --recursive`) |
| **3 (lowest)** | Built-in placeholders | Example skills bundled with the framework |

Skills at higher priority levels are never overwritten by lower-priority registrations. This means you can always override a vendor skill with your own version in the runtime mount point.

### Using the vendor submodule

If you cloned with `--recursive`, the ClawBio library is already available at `vendor/ClawBio` and all its skills will be registered automatically when the framework starts.

To update the vendor library:

```bash
git submodule update --remote vendor/ClawBio
```

### Using a custom runtime mount point

Clone or symlink the ClawBio library (or your own skill library) to the mount point:

```bash
git clone https://github.com/kzy599/ClawBio.git aqualib_workspace/skills/clawbio
pip install -r aqualib_workspace/skills/clawbio/requirements.txt
```

Skills in this directory take precedence over vendor and built-in skills.

## Usage

```bash
# List all registered skills
aqualib skills

# Run a task
aqualib run "Align the protein sequences MVKLF and MVKLT"

# Skip RAG index build (faster for testing)
aqualib run "What drugs interact with CYP2D6 poor metabolizers?" --skip-rag

# View completed tasks
aqualib tasks

# View an audit report
aqualib report <task-id>
```

## Python API

```python
import asyncio
from aqualib.bootstrap import build_orchestrator
from aqualib.config import Settings, DirectorySettings, LLMSettings

async def main():
    settings = Settings(
        directories=DirectorySettings(base="./my_workspace").resolve(),
        llm=LLMSettings(model="gpt-4o"),
        clawbio_priority=True,
    )
    orch = await build_orchestrator(settings, skip_rag_index=True)
    task = await orch.run("Align the protein sequences MVKLF and MVKLT")
    print(f"Status: {task.status.value}")

asyncio.run(main())
```

## Configuration

Create `aqualib.yaml` (generated by `aqualib init`) or set environment variables:

```yaml
llm:
  api_key: ""          # defaults to OPENAI_API_KEY env var
  model: gpt-4o
  temperature: 0.2
  max_tokens: 4096

rag:
  chunk_size: 512
  similarity_top_k: 5
  embed_model: text-embedding-3-small

clawbio_priority: true

directories:
  base: ./aqualib_workspace
```
